# Mobile App Setup

## Prerequisites

- Root `.env` filled in with `SERVER_HOST` set to your LAN IP (e.g. `192.168.100.125`)
- Docker stack running: `cd open-event-server && docker compose up -d`
- Run `python3 setup.py` from repo root to propagate `SERVER_HOST` into both mobile apps

---

## Android

**Requirements:** Android Studio, JDK 11+, Android SDK API 29+

```bash
cd open-event-organizer-android

# Check connected devices
~/Library/Android/sdk/platform-tools/adb devices

# Build debug APK
./gradlew assemblePlayStoreDebug

# Install on device/emulator
~/Library/Android/sdk/platform-tools/adb install -r \
  app/build/outputs/apk/playStore/debug/app-playStore-debug.apk
```

**Base URL** is auto-configured in `app/build.gradle` by `setup.py`. To override manually:
```java
// app/src/main/java/.../data/network/HostSelectionInterceptor.java
// or update BuildConfig.DEFAULT_BASE_URL
```

**Login credentials:** same `ADMIN_EMAIL` / `ADMIN_PASSWORD` from `.env`.

**Common issues:**

- `Cannot deserialize 'long' from String` → `@JsonIgnoreProperties("identifier")` missing on model
- `Included must be an array` → marshmallow_jsonapi bind mount not applied; rebuild OE server container
- App connects to `10.0.2.2` (emulator localhost) instead of LAN IP → re-run `setup.py`

## iOS

**Requirements:** macOS, Xcode 14+, CocoaPods

```bash
cd open-event-organizer-ios

# Install CocoaPods dependencies (first time or after Podfile change)
pod install

# Always open workspace, never .xcodeproj
open EventyayOrganizer.xcworkspace
```

Press **⌘R** in Xcode to build and run on the simulator.

**Base URL** is in `EventyayOrganizer/Helpers/LocalConfig.swift` — auto-generated by `setup.py`:
```swift
enum LocalConfig {
    static let debugBaseURL = "http://192.168.100.125:8080/v1"
}
```

**Adding new Swift files:** New files created outside Xcode must be added to the project target using the `xcodeproj` gem:
```bash
gem install xcodeproj --user-install
ruby tools/add_ios_files.rb
```

**Login credentials:** same `ADMIN_EMAIL` / `ADMIN_PASSWORD` from `.env`.

**JWT auth flow:**
1. POST `/auth/session` with email + password → `access_token`
2. Token stored in iOS Keychain via `KeychainHelper.save(key: "jwt_token", ...)`
3. All subsequent requests: `Authorization: JWT <token>`

**Common issues:**

- `No such module 'M13Checkbox'` → opened `.xcodeproj` instead of `.xcworkspace`
- Login button does nothing → old binary; rebuild with ⌘R after code changes
- Camera permission denied → `NSCameraUsageDescription` in `Info.plist` (already set)
